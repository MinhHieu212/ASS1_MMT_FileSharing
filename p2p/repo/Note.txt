Importants:

Cái Database phía server sẽ lưu file được chia sẻ như thế nào?? 
		1. Các file có trong repo của từng client (với chức năng fetch file thì phải thông báo đến server -> cập nhật lại thông tin các file có trong repo của client đó)
		2. Các file được chia sẻ chung trong hệ thống (không lưu chi tiết file đó có trong repo của client nào) -> 
		vấn đề là: mọi khi client yêu cầu tải file thì server sẽ phải Discover qua từng client cho đến khi tìm được client đang chứa file
		
Vậy là khi client chưa file cần fetch đang không hoạt động thì server sẽ return "Không có file đang được chia sẻ"



Cần chốt lại là hệ thống mình sẽ hoạt động như thế nào :

CLIENT:  

+ Chức năng fetch file thì gửi HTTP request tới Server -> tìm thông tin -> HTTP response lại cho CLient (Phúc) (NEW)

+ Việc register -> gửi dữ liệu đến BE -> lưu thông tin User (hostname , password) (Phúc)
								
+ Việc login/logout thì lấy token từ BE và validate (Phúc)

+ Chức năng upload/xóa file, upload/xóa xong -> HTTP request thông báo cho server lưu thông tin về file mới vào BD (check fname trước hay sao)  (Phúc)

+ Chức năng tìm kiếm file từ Client thì sao?? (same với chức năng fetch file) Gửi HTTP đến server -> query DB -> HTPTP client (Phúc)

+ Chức năng xem các file đang có trong hệ thống từ phía Client: gửi HTPT request tới Server để lấy thông tin này (name, size) (Phúc)

SERVER

+ Chức năng ping của Server (do admin dùng) dùng ICMP gửi đến Client check trang thái (Long)

+ Chức năng discover (do admin dùng) dùng HTTP gửi đến Client hay BE để lấy thông tin trong BD (cần chốt cách lưu thông file tin DB) (trong ảnh là kết nối đến Client ak) (Long)

+ Chức năng xem ds hostname của các client đã đăng kí tham gia vào hệ thống (hostname này chi tiết là gì) (Long)
